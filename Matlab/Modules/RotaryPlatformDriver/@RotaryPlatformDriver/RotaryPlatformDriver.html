RotaryPlatformDriver

Control interface for HTWD-DT-2025 rotary platform

Description

RotaryPlatformDriver provides a MATLAB interface to control a custom rotary platform via serial communication using the VisaIF class. The class handles all SCPI commands for position control, limit configuration, and status queries.

The device communicates at 115200 baud, 8 data bits, no parity, 1 stop bit, no flow control, with LF line terminator. Communication is fully compliant with Firmware v1.1.0 SCPI specification.

SCPI Implementation: This driver implements all 19 SCPI commands from the official HTWD-DT-2025 firmware v1.1.0 specification.

Requirements

  • VisaIF class must be in MATLAB path
  • Device must be configured in VisaIF_HTW_Labs.csv
  • Default configuration: HTWD-DT-2025 on COM7
  • MATLAB R2019b or later
  • Instrument Control Toolbox

Syntax

platform = RotaryPlatformDriver()              % Use default COM7 from config
platform = RotaryPlatformDriver('COM13')       % Override to COM13
platform = RotaryPlatformDriver({'HTWD-DT-2025', 'COM5'})  % Explicit device and port

Methods

Constructor

Method Description
RotaryPlatformDriver() Create connection using default COM7 from config file
RotaryPlatformDriver(comPort) Create connection with COM port override (e.g., 'COM13')

Device Information

Method SCPI Command Description
getID() *IDN? Get device identification string
getError() SYSTem:ERRor? Get error from error queue (clears oldest error)

Position Control

Method SCPI Command Description
setAngle(angle) ROTAtion:ANGLE Set target angle in degrees (-360 to 360)
getTargetAngle() ROTAtion:ANGLE? Query commanded target angle
getPosition() ROTAtion:POSition? Query actual current position in degrees
isReached() ROTAtion:REACHED? Check if target position is reached and held

Safety Limits

Method SCPI Command Description
setUpperLimit(angle) ROTAtion:LIMit:UPPer Set upper angle safety limit in degrees
getUpperLimit() ROTAtion:LIMit:UPPer? Query upper angle safety limit
setLowerLimit(angle) ROTAtion:LIMit:LOWer Set lower angle safety limit in degrees
getLowerLimit() ROTAtion:LIMit:LOWer? Query lower angle safety limit

Local Control

Method SCPI Command Description
lockLocal() SYSTem:LOCal:LOCK Disable physical buttons on device
unlockLocal() SYSTem:LOCal:UNLock Enable physical buttons on device
isLocked() SYSTem:LOCal:LOCK? Query if local control is locked

Motor Status

Method SCPI Command Description
isMotorEnabled() MOTOR:ENABLED? Query if motor is currently active (AND combination of all conditions)
isMotorEnableLocal() MOTOR:ENABLELOCal? Query only: Check if green enable button is pressed
isMotorEnableRemote() MOTOR:ENABLEREMote? Query remote motor enable status
setMotorEnableRemote(enable) MOTOR:ENABLEREMote Set remote motor enable (true/false) - locks motor when false
isMotorVoltLockout() MOTOR:VOLTLOCKout? Query if under-voltage lockout is active
⚠ Important: The green button state (MOTOR:ENABLELOCal?) is query-only. There is no SCPI command to remotely press the button. The button must be physically pressed on the device.
Motor Activation Logic: The motor is active when ALL of the following are true:
  • isMotorEnableLocal() == true - Green button is physically pressed
  • isMotorEnableRemote() == true - No remote lockout active
  • isMotorVoltLockout() == false - Power supply voltage is OK
isMotorEnabled() returns the AND combination of all three conditions.

Examples

Basic Connection and Movement

% Connect to device using default COM7
platform = RotaryPlatformDriver();

% Or override COM port
platform = RotaryPlatformDriver('COM13');

% Enable motor (green button must be physically pressed!)
platform.setMotorEnableRemote(true);

% Move to 90 degrees
platform.setAngle(90);
pause(2);

% Wait for position to be reached
while ~platform.isReached()
    pause(0.5);
end
disp('Position reached!');

% Get current position
pos = platform.getPosition();
fprintf('Current position: %.1f degrees\n', pos);

% Close connection
platform.delete();

Configure Safety Limits

platform = RotaryPlatformDriver();

% Query current limits
upper = platform.getUpperLimit();
lower = platform.getLowerLimit();
fprintf('Current limits: %.1f to %.1f degrees\n', lower, upper);

% Set new limits (optional)
platform.setUpperLimit(180);
platform.setLowerLimit(-180);
disp('New limits configured');

Lock Local Control During Automation

% Disable physical buttons during automated measurement
platform = RotaryPlatformDriver();
platform.setMotorEnableRemote(true);
platform.lockLocal();
disp('Physical buttons disabled');

% Perform automated measurements
for angle = 0:45:180
    platform.setAngle(angle);
    while ~platform.isReached()
        pause(0.5);
    end
    fprintf('Measuring at %.0f degrees...\n', angle);
    % ... perform measurement ...
    pause(1);
end

% Re-enable physical buttons
platform.unlockLocal();
disp('Physical buttons re-enabled');
platform.delete();

Diagnose Motor Status

platform = RotaryPlatformDriver();

% Check overall motor status
if platform.isMotorEnabled()
    disp('✓ Motor is powered and active');
else
    % Diagnose why motor is not active
    fprintf('✗ Motor is NOT active. Checking conditions:\n');
    
    localBtn = platform.isMotorEnableLocal();
    fprintf('  Green button pressed: %d ', localBtn);
    if ~localBtn
        fprintf('← PRESS GREEN BUTTON!\n');
    else
        fprintf('\n');
    end
    
    remoteEnable = platform.isMotorEnableRemote();
    fprintf('  Remote enable: %d ', remoteEnable);
    if ~remoteEnable
        fprintf('← Call setMotorEnableRemote(true)\n');
    else
        fprintf('\n');
    end
    
    voltLockout = platform.isMotorVoltLockout();
    fprintf('  Voltage lockout: %d ', voltLockout);
    if voltLockout
        fprintf('← CHECK POWER SUPPLY!\n');
    else
        fprintf('\n');
    end
end

platform.delete();

Remote Motor Lockout (Safety Feature)

platform = RotaryPlatformDriver();

% Disable motor from remote (safety feature)
% Device LCD shows lockout message, green button LED turns off
platform.setMotorEnableRemote(false);
disp('Motor locked from remote');

% Attempt to move (will not work while locked)
platform.setAngle(90);
pause(1);

% Verify motor did not move
if ~platform.isMotorEnabled()
    disp('✓ Motor did not move (as expected - remote lockout active)');
end

% Re-enable motor from remote
platform.setMotorEnableRemote(true);
disp('Motor unlocked from remote');

% Now movement will work (if green button is pressed)
if platform.isMotorEnabled()
    platform.setAngle(90);
    disp('Moving to 90 degrees...');
end

platform.delete();

Error Queue Handling

platform = RotaryPlatformDriver();

% Check for errors
err = platform.getError();
disp(['Error queue: ' err]);

% Clear all errors from queue
while ~contains(err, '0,')
    err = platform.getError();
    if ~contains(err, '0,')
        fprintf('Error cleared: %s\n', err);
    end
end
disp('Error queue is now empty');

platform.delete();
Best Practice: Always call platform.delete() or clear platform to properly close the VisaIF connection when finished. The destructor automatically cleans up resources.

Configuration

VisaIF Configuration (VisaIF_HTW_Labs.csv)

# Drehplattform-Treiber (Rotary Platform Driver)
HTWD-DT-2025 ;; HTWD ;; DrehTeller2025 ;; Other ;; visa-serial;; ASRL7::115200.8.1.none.none.LF.LF ;; 1e4 ;; 1e4

Configuration breakdown:

  • ASRL7 - COM7 (change to ASRL13 for COM13, etc.)
  • 115200 - Baud rate
  • 8.1.none.none - 8 data bits, 1 stop bit, no parity, no flow control
  • LF.LF - Line Feed terminator for both read and write
  • 1e4 - 10KB buffer sizes (input and output)

Hardware Requirements

  • HTWD-DT-2025 rotary platform with Firmware v1.1.0 or compatible
  • Serial connection (USB-to-Serial or direct COM port)
  • 12V DC power supply for motor driver
  • MATLAB R2019b or later
  • Instrument Control Toolbox
  • VisaIF class (HTW Dresden internal)

Troubleshooting

Connection Issues

Problem: "Device not found in config file"
Solution:
  • Check that HTWD-DT-2025 exists in VisaIF_HTW_Labs.csv
  • Verify CSV file is in @VisaIF folder and on MATLAB path
Problem: "No response from device"
Solution:
  • Check COM port number (use Device Manager on Windows)
  • Verify device is powered on
  • Ensure no other program is using the COM port
  • Try different COM port: RotaryPlatformDriver('COM5')

Motor Not Moving

Most Common Cause: Green button not pressed!
  • The green button MUST be physically pressed on the device
  • There is no remote command to simulate button press
  • Check: platform.isMotorEnableLocal() should return 1
Other possible causes:
  • Remote lockout active: platform.setMotorEnableRemote(true)
  • Under-voltage: Check power supply (12V DC)
  • Angle beyond limits: Check with getUpperLimit() and getLowerLimit()

Error Messages

Error: -113,"Undefined header"
Meaning: A previous SCPI command was not recognized by firmware
Action: Check firmware version compatibility. This driver requires Firmware v1.1.0.
